The application must be:

* Modular
* Scalable

* Maintainable
* Production Ready
* Well Commented

Do not generate everything in one response.

Generate **one module at a time**.

Wait for confirmation before generating the next module.

## Queue

BullMQ + Redis

Never send emails directly.

Create BullMQ jobs.

Workers must process the jobs.

Store job metadata in MongoDB.

Collection

* email_jobs

Features

* Retry
* Pause
* Resume
* Cancel
* Queue Status

---

## Scheduler

Every hour

Example

1500 contacts

Automatically send

200 emails/hour
